# IslamicAPI

Offline-first structured Islamic data SDK and reproducible dataset pipeline.

This repository is designed around one rule: religious text is never generated, inferred, or filled in by AI. Dataset records are created only by fetching public source files, validating their structure, preserving exact source text, and attaching provenance metadata.

## What It Provides

- Quran dataset pipeline and offline API helpers
- Hadith dataset pipeline and offline API helpers
- Prayer time calculation engine
- Qibla direction calculation engine
- Recitation audio indexer
- JavaScript package: `@deen/sdk`
- Python package: `deensdk`

## Install

```sh
pnpm install
pip install -e .
```

## Run The Local API

```sh
pnpm dev
```

The server listens on `http://localhost:3000` by default.

Useful endpoints:

- `GET /health`
- `GET /quran/surah/:number`
- `GET /quran/ayah/:surah/:ayah`
- `GET /quran/search?q=...`
- `GET /hadith/:id`
- `GET /hadith/random`
- `GET /prayer/times?latitude=40.7128&longitude=-74.006&date=2026-05-22&timezone=-4`
- `GET /qibla?latitude=40.7128&longitude=-74.006`
- `GET /audio/reciters`
- `GET /audio/url?reciterId=...&surah=1&ayah=1`

## Dataset Sync

```sh
pnpm discover:sources
pnpm fetch:datasets
pnpm validate:data
pnpm normalize:quran
pnpm normalize:hadith
pnpm normalize:audio
```

Or run the full autonomous pipeline:

```sh
pnpm sync:updates
```

Every fetched raw file is stored under `data/<domain>/raw/<run-id>/` with a sibling metadata file containing:

- source URL
- direct download URL
- license, when available
- retrieval timestamp
- byte size

Normalized files live under:

- `data/quran/normalized/quran.json`
- `data/hadith/normalized/hadith.json`
- `data/audio/normalized/audio-index.json`

## Source Policy

The crawler rejects or marks for review any source that is not public, has unclear redistribution terms, appears incomplete, or lacks enough provenance. Scripts do not scrape copyrighted sites without permission and do not synthesize missing text.

Seeded discovery sources include public references such as Tanzil, Quran Foundation/Quran.com API, AlQuran Cloud, Fawaz Ahmed Quran/Hadith APIs, MP3Quran, EveryAyah references, and Batoul Apps Adhan calculation libraries. The registry is regenerated by `pnpm discover:sources`.

## JavaScript Usage

```js
import {
  calculatePrayerTimes,
  getAyah,
  getQiblaDirection,
  getSurah,
  search
} from "@deen/sdk";

const surah = getSurah(1);
const ayah = getAyah(1, 1);
const results = search("query");
const qibla = getQiblaDirection(40.7128, -74.006);
const times = calculatePrayerTimes({ latitude: 40.7128, longitude: -74.006, timezone: -4 });
```

## Python Usage

```py
from deensdk import calculate_prayer_times, get_ayah, get_qibla_direction

ayah = get_ayah(1, 1)
qibla = get_qibla_direction(40.7128, -74.006)
times = calculate_prayer_times(latitude=40.7128, longitude=-74.006, timezone=-4)
```
