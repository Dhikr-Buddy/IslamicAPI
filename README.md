# IslamicAPI

Offline-first structured Islamic data SDK and reproducible dataset pipeline.

This repository is designed around one rule: religious text is never generated, inferred, or filled in by AI. Dataset records are created only by fetching public source files, validating their structure, preserving exact source text, and attaching provenance metadata.

## What It Provides

- Quran dataset pipeline and offline API helpers
- Hadith dataset pipeline and offline API helpers
- Prayer time calculation engine
- Qibla direction calculation engine
- Recitation audio indexer
- Mushaf font serving for Uthmani, Naskh/Indopak, and plain Arabic display
- JavaScript package: `@deen/sdk`
- Python package: `deensdk`
- Static Web API through GitHub raw links

## Dataset Sync

```sh
pnpm discover:sources
pnpm fetch:datasets
pnpm validate:data
pnpm normalize:quran
pnpm normalize:hadith
pnpm normalize:audio
pnpm normalize:fonts
pnpm build:api-index
```

Or run the full autonomous pipeline:

```sh
pnpm sync:updates
```

Every fetched raw file is stored under `data/<domain>/raw/<run-id>/` with a sibling metadata file containing:

- source URL
- direct download URL
- license, when available
- retrieval timestamp
- byte size

Normalized files live under:

- `data/quran/normalized/quran.json`
- `data/hadith/normalized/hadith.json`
- `data/audio/normalized/audio-index.json`
- `data/fonts/normalized/fonts.json`
- `data/api/index.json`

## Source Policy

The crawler rejects or marks for review any source that is not public, has unclear redistribution terms, appears incomplete, or lacks enough provenance. Scripts do not scrape copyrighted sites without permission and do not synthesize missing text.

Seeded discovery sources include public references such as Tanzil, Fawaz Ahmed Quran/Hadith APIs, MP3Quran, EveryAyah references, and Batoul Apps Adhan calculation libraries. Sources with unclear redistribution terms are either indexed with explicit provenance or rejected from text mirroring. The registry is regenerated by `pnpm discover:sources`.

## Current Synced Dataset

- Quran: 6,236 ayah records from a Tanzil-backed public package, with source metadata.
- Hadith: full Arabic/English available records for Bukhari, Muslim, Abu Dawud, Tirmidhi, Nasai, and Ibn Majah from the public Fawaz Ahmed hadith API dataset.
- Audio: MP3Quran surah-level URL indexes and EveryAyah ayah-level URL templates, with ZIP/MD5 references where published.
- Fonts: local WOFF2 assets for Uthmani variants, Naskh/Indopak-style variants, and plain Arabic fallbacks.
- Prayer/Qibla: offline calculations only.

## JavaScript Usage

```js
import {
  calculatePrayerTimes,
  createRawDataClient,
  getAudioUrl,
  getAyah,
  getFontList,
  getQiblaDirection,
  getSurah,
  search
} from "@deen/sdk";

const surah = getSurah(1);
const ayah = getAyah(1, 1);
const results = search("query");
const qibla = getQiblaDirection(40.7128, -74.006);
const times = calculatePrayerTimes({ latitude: 40.7128, longitude: -74.006, timezone: -4 });

const raw = createRawDataClient();
const index = await raw.loadIndex();
```

## Python Usage

```py
from deensdk import calculate_prayer_times, get_ayah, get_font_list, get_qibla_direction, github_raw_url

ayah = get_ayah(1, 1)
fonts = get_font_list()
qibla = get_qibla_direction(40.7128, -74.006)
raw_index = github_raw_url("data/api/index.json")
times = calculate_prayer_times(latitude=40.7128, longitude=-74.006, timezone=-4)
```

## Browser Usage

The GitHub Pages wiki includes a tiny ESM client:

```js
import { createDeenRawClient } from "https://dhikr-buddy.github.io/IslamicAPI/deen-web.js";

const deen = createDeenRawClient();
const quran = await deen.loadQuran();
const hadith = await deen.loadHadith();
```
